## 🎯 功能介绍
这是一个跨平台外卖比价点单工具，支持美团、饿了么双平台价格对比，自动推荐最优选择，减少点单决策时间。

### ✅ 已实现功能
1. **多平台比价**：同时查询美团、饿了么商家价格，自动排序推荐
2. **优惠券自动计算**：自动展示可用优惠券，计算最优优惠后价格
3. **口味记忆**：支持添加忌口食材，自动过滤不符合偏好的商家
4. **历史订单记录**：记录点单历史，下次优先推荐常吃菜品
5. **快速跳转**：直接提供商家点单链接，无需手动搜索
6. **位置适配**：支持自定义位置查询附近商家

## 🚀 使用方法
### 1. 最终一键版（推荐）
```bash
cd fanlai
python final.py 想吃的东西 [位置（可选）]
```
示例：
```bash
python final.py 米线 上海虹桥火车站
```
直接返回最优选择和付款链接，点进去直接付款。

### 2. 完整功能版
```bash
python full_version.py
```
进入菜单选择：
- 1. 点单比价
- 2. 查看个人配置
- 3. 添加忌口食材
- 4. 查看历史订单

### 3. 快速比价版
```bash
python real_mvp.py
```
直接输入想吃的，快速对比价格。

### 4. 极简版
```bash
python fanlai.py
```
自动显示优惠券，按预算推荐套餐。

## 🛠️ 核心逻辑说明
### 比价算法
1. 同时爬取美团、饿了么两个平台的商家数据
2. 计算综合成本：餐品价格 + 配送费 - 最大可用优惠
3. 按综合成本从低到高排序，优先推荐性价比最高的
4. 同时考虑配送时间、商家评分等因素，给出综合最优

### 优惠计算逻辑
1. 自动识别平台通用优惠券、商家专属优惠券、满减活动
2. 计算所有可用优惠的最优组合
3. 直接显示最终实付价格，不需要用户自己计算

### 偏好过滤逻辑
1. 用户添加的忌口食材会永久保存在配置文件中
2. 搜索时自动过滤包含忌口食材的商家和餐品
3. 记录用户常吃的商家和菜品，下次优先推荐

## 🔧 技术方案
### 数据获取方案
- 放弃美团官方API（仅对企业开放，个人无法申请）
- 采用浏览器自动化方案（Playwright）爬取公开数据
- 模拟用户正常请求，不违反平台使用规则
- 数据仅本地使用，不上传到任何服务器

### 技术栈
- 核心语言：Python 3.10+
- 爬虫框架：Playwright / Requests
- 数据存储：JSON文件（本地存储，不涉及云端）
- 界面：命令行工具（后续可扩展为Web/小程序）
